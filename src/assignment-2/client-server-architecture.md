# Client-Server Architecture, Express.js, Frameworks, and REST APIs

## 1. What is Client-Server Architecture?

Client-server architecture is a fundamental concept in web development. It describes how two main components — clients and servers — communicate with each other.

### Definitions:
- **Client**: A device or program (like a browser or mobile app) that requests data or services.
- **Server**: A machine or application that processes those requests and sends back a response.

### Example:
When you visit `www.google.com`:
1. Your browser (client) sends a request to Google’s server.
2. The server processes the request and returns the search page as a response.

This architecture ensures that:
- The client handles UI and user input.
- The server handles logic, database access, and decision-making.

---

## 2. What is Express.js?

Express.js is the most widely used framework for building web servers and REST APIs in Node.js.

It helps developers build backend applications quickly and efficiently by providing a clean structure and reusable tools.

### Features of Express.js:
- Built on top of Node.js
- Simplifies HTTP request handling (GET, POST, etc.)
- Supports middleware (custom functions that process requests)
- Ideal for building RESTful APIs
- Handles routing, error handling, static file serving, and more
